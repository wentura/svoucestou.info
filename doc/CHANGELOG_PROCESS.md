# Changelog process

## Co zapisovat
- Změny produkčního chování.
- Změny SEO, bezpečnosti, routingu a cen/kontaktů.
- Hotfixy a rollbacky.

## Formát záznamu
- Datum
- Typ změny (feature/fix/docs/security/hotfix)
- Krátké proč
- Dopad na uživatele/provoz
- Odkaz na PR/commit

## Release summary
Po každém release vytvoř krátké shrnutí:
1. Co se měnilo.
2. Co bylo ověřeno.
3. Otevřená rizika a další kroky.
